# Icons

SVG-based icons for use in the design system.

There are some exceptions to this, but the majority of the icons are 24x24px by default.

Icons may contain up to three different color markers: 
 - `main` - basic color of the svg used everywhere
 - `alt` - for some parts that should be differenly colored than the main.
 - `accent` - for parts needs to be highlithed.

In the component it's just data-attribute associated with the correcponding part of the icon.

<Subtitle>Example of color data attributes</Subtitle>
<Source code={`
function SvgFilterActive({ ...props }) {
  return (
    <svg xmlns="http://www.w3.org/2000/svg" width={24} height={24} fill="none" viewBox="0 0 24 24" ref={ref} {...props}>
      <path
        fill="currentColor"
        data-color="main"
      />
      <path fill="currentColor" data-color="accent" />
    </svg>
  );
}
`}  dark/>


## Installation

<Source code={`npm i @giosg/design-system-icons`} language="bash" dark />

## Usage 

Import icons you want to use and use it as a component.

<Subtitle>Batch import</Subtitle>
<Source code={`
import * as Icons from "@giosg/design-system-icons";

export const App = () => {
    return (
      <Icons.ArrowRight />
    );
  },
};
`}  dark/>

<Subtitle>Individual import</Subtitle>
<Source code={`
import {AiChatbot} from "@giosg/design-system-icons";

export const App = () => {
    return (
      <AiChatbot />
    );
  },
};
`}  dark/>



<Subtitle>Usage without foundation package</Subtitle>
Icons are completely consumable without foundation package. I want to highlight some of th key points and simplifications that are made for that.

Bu default main and accent parts of the svgs are `fill=currentColor` so in most cases you don't need to do anything.
In case when you need to changes that color use any css solutions you are currently using and target that parts of svg just like that.
<Source code={`
.icon-css {
  [data-color="accent"] {
    fill: beige;
  }

  [data-color="main"] {
    fill: black;
  }

  [data-color="alt"] {
    fill: white;
  }
}
`} language="css" dark/>

<Subtitle>Usage with foundation package</Subtitle>

In foundation packages the data attributes are automatically assigned to the corresponding parts of the svg. So icons are automatically changing the theme along with the rest of the app.
<Source code={`
:root,
[data-theme="light"],
[data-theme="dark"] {
  --gds-svg-main: var(--gds-sys-color-fg-neutral-default);
  --gds-svg-alt: var(--gds-sys-color-fg-neutral-inverted);
  --gds-svg-accent: var(--gds-sys-color-fg-primary-strong);
}

[data-color="main"][fill] {
  fill: var(--gds-svg-main);
}

[data-color="alt"][fill] {
  fill: var(--gds-svg-alt);
}

[data-color="accent"][fill] {
  fill: var(--gds-svg-accent);
}
`} language="css"  dark/>

If you need to adjust or change the value you are absolutely free to do that at any level of your app. Just chagne the `--gds-svg-main`, `--gds-svg-alt`, `--gds-svg-accent` to shadow outer-scoped variable. Just the way its done in the main story for the icons. For inspiration go there and try to play with the controls of the story.
